# raceGPS Akron Beta — Unreal Engine 5

Commercial-grade driving slice for Akron, Ohio. Built on **Unreal Engine 5.5** with **Chaos Vehicles**, **World Partition**, and **OpenDRIVE** semantic ingestion.

## Quick Start

### Prerequisites

- Unreal Engine 5.5 (via Epic Games Launcher or source)
- Visual Studio 2022 with C++ game dev workload
- Python 3.10+ (for semantic compiler)

### Build

1. **Generate project files**:
   ```powershell
   # From the project root
   .\raceGPSAkronBeta.uproject   # UE5 will prompt to generate VS files
   # OR
   .\GenerateProjectFiles.bat    # If using source engine
   ```

2. **Build in Visual Studio**:
   - Open `raceGPSAkronBeta.sln`
   - Set configuration to **Development Editor**
   - Build `raceGPSAkronBeta` project

3. **Open in UE5 Editor**:
   - Double-click `raceGPSAkronBeta.uproject`
   - Load `Content/Maps/AkronWorld`

### Run

- **Editor Play**: PIE (Play In Editor) with `AkronWorld` open
- **Standalone**: Package via `File → Package Project → Windows`
- **Development**: `./Binaries/Win64/raceGPSAkronBeta-Win64-Shipping.exe`

## Game Mode: Cruise Sprint

1. **Loading** — Import city data (roads, routes, spawns, POIs)
2. **Countdown** — 3-2-1-GO timer
3. **Racing** — Drive through checkpoints, timer running
4. **Finished** — Display time + medal (Gold / Silver / Bronze)

### Medal Times

| Medal | Time |
|-------|------|
| Gold | ≤ 120s |
| Silver | ≤ 150s |
| Bronze | ≤ 200s |

## Data Contract

The game loads three data sources at runtime:

1. **`manifest.json`** — City metadata, spawn points, POIs, route list
2. **`akron.xodr`** — OpenDRIVE road network (XML)
3. **`routes/*.json`** — Per-route waypoint splines + checkpoint gates

All coordinates in the JSON are **WGS84 (lat, lon)**. The `AkronXodrImporter` converts them to Unreal world space using the origin from `manifest.json`.

## Development Notes

- **Vehicle physics**: Chaos Vehicles (not PhysX). Substepping enabled at 60Hz.
- **World streaming**: World Partition with 500m cells. Roads load/unload based on vehicle position.
- **Road rendering**: Currently stubbed. Final implementation should generate procedural meshes from XODR geometry or pre-bake static meshes.
- **XODR parser**: `AkronXodrImporter::ImportXodr` is a stub. Integrate CARLA `Osm2Odr` or write a proper XML parser.
